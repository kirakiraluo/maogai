表单

表单包含了很多主要的前后端交互(用户操作数据向后端发送处理).

需要做到的事情有:

  1. 实现表单的视图(一些基本种类,一些可以使用的装饰,一些可选选项)
  2. 使用js等方法处理操作(验证,向后台提交(post or get))

视图层的表单

<form action="func" method="POST or GET">
	<div>
		<label for="test1"></label>
		<input type="text" id="test1" name="input1">
	</div>
	<div>
		<label for="test2"></label>
		<input type="text" id="test2" name="input2">
	</div>
</form>

在浏览器中定义一个表单域, 可以指定action(用户点击提交后触发的事件),method(get or post).

可以将label与对应的input放在一个div内,方便组织与相关样式调整.

label标签

上面首先定义了一个label标签,标签内有一个for属性.

for属性可以让label与相关的input产生关联.

点击一个label中的文字(或者区域),连接的input框被选中(产生其他操作)

label可以不定义在表单内,而且for的对象只要是id就行,甚至可以绑定div等等,但是没有效果.

使用方式有两种:

// method1
<label>string<input type="..." ...>string</label>

//method2
<label for="test1">string</label>
<input type="..." id="test1">

第一种是隐式说明,不需要for,input也不需要id.

第二种则是显式,更加灵活.

推荐使用显示.

一个input是可以绑定多个label的.但是这样会很奇怪,正确的方式是使用隐式写入,然后在input前后加上说明.(然后加上for与id).

input标签

input可以说是表单中最重要的标签了.

我们可以使用type用来表示不同的内容.

常见的有:

  1. text: 通常是一行文字输入框,可以使用value="string"属性预先填充值.

与之相对的是textarea, 这是个标签;<textarea>string</textarea>,string填入预先默认填充的值.

textarea方便大规模的填充文本.

  1. date: 时间,提供一个时间的选择.
  2. color: 提供一个调色盘来进行颜色选择.
  3. checkbox: 多选框(方形).
  4. radio:单选框(圆形).
  5. password:密码输入框.
  6. submit:实则为一个按钮(button),在form里面点击按钮就能触发form事件.类似text,使用value可以修改显示在按钮上的文字.

目前有一个专门的button组件,使用<button>string</button>可以方便的调节文字,样式.

button 也能设定type,一般有三种:

  1. submit(默认):提交,触发事件.
  2. reset: 将表单重置为默认值.
  3. button: 啥也不做,一般用来js绑定其他事件.

此外,button支持完整html元素(可以插入图片等等),而使用input创建的提交只能显示文字.

一些常用的widgets(小组件)

以前使用python原版tkinter的时候基本不会灵活使用widgets,只会简单的输入,单选复选.后来才知道其实很多gui都内置了很多很好用(默认样式丑)的小组件(比如选择文件,下拉栏目等等).

组件的通用属性:

组件有一些通用属性,便于表单提交的规范:

  1. disabled: 禁用组件,这个元素的值也不会随着其他数据一起发送

  2. readonly: 禁止用户修改,但是数据会一同发送.

    上面都是一个属性,可以通过js添加删除.

  3. placeholder: 灰色背景提示

  4. form:相关联的表单元素

  5. name: 提交到表单内的变量名称

  6. value: 元素的初始值

哪些可以被称作widgets

input: 文本输入框

select:下拉框

<select id="" name="simple">
	<option>name</option>
	<option selected>name2</option>
</select>

添加选项,这样就能为select填入值(默认为option内部的值,也可添加value来传递值).

checkbox:复选框

<input value="dsas" name="test" type="checkbox">
<input value="dsads" name="test" type="checkbox">
<input value="dsads" name="test1" type="checkbox">

共用同一个name的input会被绑定在同一个值上.

radio:单选框

<ul>
    <li>
      <label for="soup">Soup</label>
      <input type="radio" checked id="soup" name="meal" value="soup">
    </li>
    <li>
      <label for="curry">Curry</label>
      <input type="radio" id="curry" name="meal" value="curry">
    </li>
    <li>
      <label for="pizza">Pizza</label>
      <input type="radio" id="pizza" name="meal" value="pizza">
    </li>
  </ul>

一个很好的办法是放置在一个ul表单里,在li中著名标签与单选框.

只要共用同一个name,那么它们绑定的就是一个值.选择其中一个会取消对其他的选择.

单复选都可以用checked来初始默认选择值.

滑块:

<input type="range" name="beans" id="beans" min="0" max="500" step="10">

定义一个可以滑动的滑块,定义了name,最值,步长.

日期时间选择

色彩选择(很丑,但是实用)

文件选择:

<input type="file" name="file" id="file" accept="image/*" multiple>

选择文件上传,accept接受一个MIME类型.

会显示你提交的文件名称.这一般很丑.

如果是提交图片的话,我们需要上传后得到预览,但是原来的选择器太丑了.

这一般是需要js配合使用,具体大致为文件上传得到url,然后作为img的src来显示,原本文件的选择则隐藏.

.inputfile {
            opacity: 0;
            position: absolute;
        }

<input type="file" name="file" id="file" class="inputfile" />
<label for="file" class='btn btn-success'>Choose a file</label>

让实际的文件选择器脱离文档流并不可见.让选择的内容落到label上(可以调节为一个我们满意的btn).

这样用户点击的上传按钮实际上是label,原本的input是不可见的.

隐藏内容:

<input type="hidden" id="timestamp" name="timestamp" value="1286705410">

与disabled, readonly不同,元素是不可见的,但是这个值能被提交.需要设定默认值.

form表单

使用<form action="" method="GET" ></form>就能创建表单域.

上面的action表示触发对应的js事件,method指定方法.

为了得到对应表单元素的引用,需要为每个input指定一个name属性.

<input name="user_name" type="text">

这样提交的时候就能通过name(上面为user_name)属性得到输入的内容.

一张表单可能很长(比如有很多问题的调查问卷),它们共用一个submit.

一般来说,可以根据题目类型分类,这样可以更加语义化.

可以使用下面标签做到:

<form>
	<fieldset>
		<legend>info</legend>
		<label>string<input ...></label>
		<label>string<input ...></label>
		<label>string<input ...></label>
		<label>string<input ...></label>
	<fieldset>
	<fieldset>
		<legend>info</legend>
		<label>string<input ...></label>
		<label>string<input ...></label>
		<label>string<input ...></label>
		<label>string<input ...></label>
	<fieldset>
</form>

每一个fieldset块会将内部元素加上默认黑色边框,并且可以使用legend标签加上标题.

实际上也可以使用div,然后自己调节css来修饰表单(加上边框等等).

fieldset的好处是给我们一个很好的内联标题legend.(当然对于某强迫症还是喜欢自己改).

这样的好处是方便语义化.

还有一种语义化的方法就是在标签上指定title="string",这样在鼠标悬浮的时候会弹出默认提示信息.

自定义组件样式

原本的组件基本都很丑,需要使用css等手段美化.

一些方便美化的:

button: 可以使用大量css,还可以使用js来修饰.

一些不方便美化的:

颜色选择,文件选择,多选复选.

但是总有可以取巧的方法.

比如下拉选择完全可以自己写一个ul的下拉栏,然后选择对应的值.

对于一些其他的,往往可以使用label连接对应的组件,然后隐藏原来丑陋的组件,从而修改label(往往自由度很高,但是需要花费大量时间来学习调试效果).

数据发送

使用form的action发送数据:

action可以指定为绝对url,从而向绝对url发送对应方法的数据.

也可是相对url.

还可是函数,让js来发送.

这样的好处是可以通过js来返回验证情况,并对参数做进一步处理(比如加密,判断等等).

get与post

其实两者差别不是很大.

get方法主要是将数据作为参数url/?paraName=value&para2Name=value2来发送.

post用来发送内容比较多的数据.数据附到FormData里.

数据验证

验证可以直接在html中完成,也可在js中完成.

在html中:

使用属性pattern="re"引入一个正则表达式来判断.

还可以限制长度等等

但是缺点很明显:

提示框是基于浏览器的,无法修改它们的样式.

所以js处理才是真正有效的方法.

在js中:

有很多api可以直接调用来处理一些常见的验证.

为使用js验证,首先需要关闭表单的验证:

<form novalidate>
</form>

然后使用js来为表单添加事件倾听.

// 有许多方式可以获取 DOM 节点；在此我们获取表单本身和
// email 输入框，以及我们将放置错误信息的 span 元素。

var form  = document.getElementsByTagName('form')[0];
var email = document.getElementById('mail');
var error = document.querySelector('.error');

email.addEventListener("input", function (event) {
  // 当用户输入信息时，校验 email 字段
  if (email.validity.valid) {
    // 如果校验通过，清除已显示的错误消息
    error.innerHTML = ""; // 重置消息的内容
    error.className = "error"; // 重置消息的显示状态
  }
}, false);
form.addEventListener("submit", function (event) {
  // 当用户提交表单时，校验 email 字段
  if (!email.validity.valid) {

    // 如果校验失败，显示一个自定义错误
    error.innerHTML = "I expect an e-mail, darling!";
    error.className = "error active";
    // 还需要阻止表单提交事件，以取消数据传送
    event.preventDefault();
  }
}, false);

这样提交按钮其实权力也被移交给了js.

我们可以自定义提示错误框(使用js在特定位置改变样式,生成等等).

常用的办法是添加一个类
